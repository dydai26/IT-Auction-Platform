# Документація проекту "IT Аукціони"

Цей файл створено для фіксації всієї архітектури, дизайну та функціоналу проекту. Він слугує орієнтиром для подальшої розробки та підтримки.

## 💻 Стек технологій
- **Мова програмування**: TypeScript
- **Фреймворк**: Next.js (App Router, React)
- **База даних та Авторизація**: Supabase (PostgreSQL, Supabase Auth)
- **Стилізація**: Vanilla CSS (CSS Modules) та глобальні CSS змінні (`globals.css`)

## 🌍 Багатомовність (i18n)
Проект підтримує 3 мови:
- **RU** (Російська - за замовчуванням)
- **EN** (Англійська)
- **ZH** (Китайська)

Логіка реалізована через `Next.js Middleware` (який перенаправляє на відповідний `/[lang]` роут) та клієнтський `LanguageContext`.
Словники зберігаються у `src/lib/locales/ru.ts`, `en.ts`, `zh.ts`.

## 🎨 Дизайн та Інтерфейс
Проект використовує мінімалістичний та преміальний стиль із золотими акцентами.

### Кольорова палітра
- **Основний акцент (Gold)**: `#d4af37` (використовується для ховер-ефектів, логотипу, підкреслень).
- **Фон сайту**: Переважно світлий (`#ffffff`, `#f9fafb`, `#f8f9fa`).
- **Текст**: Темні відтінки для контрасту (`#171717`, `#111827`, `#4b5563`).
- **Статуси лотів**:
  - *Активний*: Жовтий фон (`#fef08a`), темно-золотий текст (`#854d0e`).
  - *Завершений*: Червоний фон (`#fecaca`), темно-червоний текст (`#991b1b`).

### Шрифти
- Використовуються сучасні системні шрифти через `next/font` (Geist Sans, Geist Mono).
- Ваги шрифтів: `400` (Regular), `500` (Medium), `600` (SemiBold), `700` (Bold).

## 🚀 Основний Функціонал
1. **Каталог аукціонів (`/auctions`)**
   - Відображення карток лотів (фото, назва, ціна, статус, час до завершення).
   - Фільтрація та пошук (у планах/частково).

2. **Сторінка лоту (`/auction/[id]`)**
   - Детальний опис, поточна ціна, форма для ставки.
   - Історія ставок (відображається анонімне ім'я користувача, а для власної ставки підсвічується "(Ви)").
   - Ставки не обмежені кількісно і не мають мінімального кроку (можна ввести будь-яку суму, вищу за поточну).

3. **Авторизація (Supabase Auth)**
   - Модальне вікно (Увійти / Реєстрація) з полями: Ім'я, Email, Телефон, Пароль.
   - Дані про сесію зберігаються в cookies (за допомогою `@supabase/ssr`).
   - *Тестовий доступ*: Логін `admin`, пароль `123456` (створює локальну mock-сесію адміністратора).

4. **Адмін-панель (`/admin`)**
   - Доступ обмежено через `middleware.ts` лише для користувачів з `role === 'admin'`.
   - **Dashboard**: Загальна статистика.
   - **Lots Management**: Створення, редагування, видалення лотів, перегляд історії ставок.
   - **Categories**: Додавання та видалення категорій лотів.
   - **Users**: Перегляд списку зареєстрованих користувачів.
   - **Settings**: Налаштування сповіщень (Email, SMS, Telegram).
   - *Примітка*: До підключення реальної БД Supabase, адмінка працює в "тестовому режимі" через `localStorage` та `AdminTestContext`.

## 📁 Структура папок (основна)
- `src/app/[lang]/` - Основні сторінки сайту (Home, /auctions, /auction/[id], /admin).
- `src/components/` - Перевикористовувані UI компоненти (Header, Footer, AuctionCard, AuthModal).
- `src/context/` - Провайдери стану (LanguageContext, AdminTestContext).
- `src/lib/` - Утиліти, налаштування клієнта/сервера Supabase, словники перекладів.
- `supabase/migrations/` - SQL скрипти для розгортання бази даних.
